3. Constant

상수. 값을 할당 뒤 변경불가 = immutable

왜 사용?

-> 보안상의 이유. 외부요인(해킹)에 의해 값이 변경되지 않게.

-> Thread safety. 여러Thread가 접근하는 연산의 경우, 값 일관성

-> 개발시 실수(human mistake) 방지

4. Variable Types

primitive type : 더이상 작은 단위로 나뉘어질 수 없는, single item

-> number, string, boolean, null, undifined, symbol

Object type : single item을 묶어서 한 박스로, 한 단위로 관리하게 해주는 타입

JS에서는 function도 데이터 타입중하나 -> first-class function

-> 변수에 할당 가능하고, 인자로 전달하거나 리턴에 넣을 수 도 있다.

-> 할당할 값의 크기를 미리 생각해서 형을 지정해줘야 메모리 낭비 방지가능

symbol

식별자(유니크)가 필요한 경우

const symbol1 = Symbol('id');

const symbol2 = Symbol('id');

symbol1 === symbol2 -> false

동일한 string을('id') 사용였지만, Symbol은 고유하므로 서로 다름.

...동일한 string을 사용하고 싶다면

const symbol1 = Symbol.for('id');

const symbol2 = Symbol.for('id');

symbol1 === symbol2 -> true
